agent: plan
---

把用户需求转成可执行计划，要求：可审计、可验证、可回滚。

## 输入
$ARGUMENTS

## 输出结构（强制）
1) 目标（1 句话）
2) 非目标（防止 scope creep）
3) 方案概述（1~3 条）
4) 任务拆解（最多 7 步，每步都有可验证输出）
5) 风险点与缓解（至少 3 条）
6) 验证命令清单（按项目类型选择）
   - 后端：`uv run python -m pytest`
   - 前端（Ant Design Pro）：`yarn lint`、`yarn build`
7) 回滚策略（如何撤回到上一个可用状态）

## 规则
- 缺少关键信息必须先问，不允许瞎猜。
